# Session 3 — Atelier

## Livrable

Vous devez rendre **deux éléments** :

1. **Les wireframes**
2. **Les fichiers HTML** de chaque page, **sans CSS**

## Pages à réaliser

Vous devez produire les wireframes et le HTML pour les pages suivantes :

* **Home**
* **Contact**
* **Catalog**
* **Item**

# 1. Wireframes

Pour chaque page, réalisez au minimum les versions suivantes :

* **Petit mobile**
* **Petite tablette**
* **Desktop**

Les wireframes doivent montrer clairement :

* la structure générale de la page
* la hiérarchie des contenus
* les zones de navigation
* les blocs principaux
* les éléments interactifs essentiels

# 2. Intégration HTML

Pour chaque page, fournissez un fichier HTML **sans CSS**.

Le HTML doit respecter les critères suivants :

* **sémantique**
* **accessible**
* **valide**
* conforme aux principes **WCAG**

## Attendus techniques

Votre code doit notamment :

* utiliser les balises sémantiques appropriées (`header`, `nav`, `main`, `section`, `article`, `aside`, `footer`, etc.)
* respecter une structure logique des titres (`h1`, `h2`, `h3`, etc.)
* associer correctement les `label` aux champs de formulaire
* prévoir des intitulés explicites pour les liens, boutons et champs
* être lisible, correctement indenté et organisé
* fonctionner sans dépendre du CSS pour être compris
* passer la validation HTML
* appliquer les bonnes pratiques d’accessibilité et de structure WCAG

# Focus particulier par page

## Catalog

Pour la page **Catalog**, le travail doit mettre l’accent sur :

* **les cartes de contenu**
* **les filtres de recherche**

Les wireframes et le HTML doivent donc montrer clairement :

* une zone de filtres identifiable et bien structurée
* des champs de recherche et/ou de tri organisés de manière logique
* des cartes répétées de façon cohérente
* une hiérarchie claire dans chaque carte : titre, image éventuelle, informations principales, lien ou action
* une structure HTML adaptée aux listes de résultats et aux contrôles de filtrage

Une attention particulière est attendue sur :

* la sémantique des formulaires de filtre
* l’accessibilité des champs, cases à cocher, boutons radio, listes déroulantes ou boutons
* la structure des cartes, qui doit rester compréhensible sans mise en forme CSS

# Récapitulatif du rendu

Pour chaque page (**Home, Contact, Catalog, Item**), vous devez fournir :

* **3 wireframes minimum** : petit mobile, petite tablette, desktop
* **1 fichier HTML** sans CSS

# Objectif de l’atelier

Cet atelier vise à :

* concevoir une interface adaptée à plusieurs formats d’écran
* structurer correctement une page web avant le travail visuel
* produire un HTML propre, sémantique et accessible
