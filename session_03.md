# Session 3 — Atelier
Durée : **4h**

Cette séance a pour objectif de transformer les premières intentions de conception en une **base exploitable pour la suite du projet**.  
Vous devez produire à la fois une représentation claire des pages et une première intégration HTML propre, sans travail visuel.

---

## Livrables

Vous devez rendre **deux éléments** :

1. **Les wireframes**
2. **Les fichiers HTML** de chaque page, **sans CSS**

---

## Pages à réaliser

Vous devez produire les wireframes et le HTML pour les pages suivantes :

- **Home**
- **Catalog**
- **Item**

---

# 1. Wireframes

Pour chaque page, réalisez au minimum les versions suivantes :

- **Petit mobile**
- **Petite tablette**
- **Desktop**

Les wireframes doivent montrer clairement :

- la structure générale de la page
- la hiérarchie des contenus
- les zones de navigation
- les blocs principaux
- les éléments interactifs essentiels
- le parcours de lecture et d’usage principal de la page

Un wireframe doit permettre de comprendre la logique de la page, même sans style graphique final.

---

# 2. Intégration HTML

Pour chaque page, fournissez un fichier HTML **sans CSS**.

Le HTML doit respecter les critères suivants :

- **sémantique**
- **accessible**
- **valide**
- conforme aux principes **WCAG**
- compréhensible **sans mise en forme CSS**

Le site peut être visuellement brut à ce stade.  
Ce qui compte ici, c’est la qualité de la structure, la clarté du contenu et la cohérence du balisage.

---

## Attendus techniques

Votre code doit notamment :

- utiliser les balises sémantiques appropriées (`header`, `nav`, `main`, `section`, `article`, `aside`, `footer`, etc.)
- respecter une structure logique des titres (`h1`, `h2`, `h3`, etc.)
- associer correctement les `label` aux champs de formulaire
- prévoir des intitulés explicites pour les liens, boutons et champs
- être lisible, correctement indenté et organisé
- fonctionner sans dépendre du CSS pour être compris
- passer la validation HTML
- appliquer les bonnes pratiques d’accessibilité et de structure WCAG
- conserver une structure cohérente d’une page à l’autre
- préparer une base propre pour le futur travail CSS et PHP

---

# Focus particulier par page

## Home

La page **Home** doit permettre d’identifier immédiatement :

- le rôle du site
- les contenus ou accès principaux
- la navigation essentielle
- les zones qui introduisent le reste du projet

Une attention particulière est attendue sur :

- la hiérarchie de l’information
- la clarté du contenu principal
- l’accès aux autres pages importantes

---

## Catalog

Pour la page **Catalog**, le travail doit mettre l’accent sur :

- **les cartes de contenu**
- **les filtres de recherche**

Les wireframes et le HTML doivent donc montrer clairement :

- une zone de filtres identifiable et bien structurée
- des champs de recherche et/ou de tri organisés de manière logique
- des cartes répétées de façon cohérente
- une hiérarchie claire dans chaque carte : titre, image éventuelle, informations principales, lien ou action
- une structure HTML adaptée aux listes de résultats et aux contrôles de filtrage

Une attention particulière est attendue sur :

- la sémantique des formulaires de filtre
- l’accessibilité des champs, cases à cocher, boutons radio, listes déroulantes ou boutons
- la structure des cartes, qui doit rester compréhensible sans mise en forme CSS

---

## Item

La page **Item** doit permettre de comprendre clairement :

- comment un contenu complet est affiché
- quelles informations sont prioritaires
- quelles actions ou navigations secondaires sont possibles

Une attention particulière est attendue sur :

- la hiérarchie du contenu
- la clarté du titre principal
- l’organisation des informations détaillées
- la lisibilité de la page sans mise en forme CSS

---

# Critères de validation

La séance est considérée comme réussie si :

- les wireframes des 3 pages demandées existent
- chaque page existe aussi en HTML
- les wireframes montrent bien l’adaptation aux 3 formats demandés
- la structure générale des pages est compréhensible
- la hiérarchie des contenus est visible
- la navigation est identifiable
- le HTML est propre, logique et lisible
- la page **Catalog** montre clairement filtres et cartes
- les pages peuvent être comprises sans dépendre du CSS

---

# Récapitulatif du rendu

Pour chaque page (**Home, Catalog, Item**), vous devez fournir :

- **3 wireframes minimum** : petit mobile, petite tablette, desktop
- **1 fichier HTML** sans CSS

Au total, cela représente donc :

- **9 wireframes minimum**
- **3 fichiers HTML**

---

# Objectif de l’atelier

Cet atelier vise à :

- concevoir une interface adaptée à plusieurs formats d’écran
- structurer correctement une page web avant le travail visuel
- produire un HTML propre, sémantique et accessible
- préparer une base stable pour le travail CSS des séances suivantes
- commencer à homogénéiser les structures entre les différentes pages du projet

---

# Checklist de fin de séance

Avant de quitter la séance, vérifiez que vous avez bien :

- réalisé les wireframes de **Home**, **Catalog** et **Item**
- décliné chaque page en **petit mobile**, **petite tablette** et **desktop**
- créé les fichiers HTML correspondants
- mis en place une structure sémantique cohérente
- vérifié la hiérarchie des titres
- rendu les formulaires compréhensibles et correctement étiquetés
- structuré clairement la zone de filtres de **Catalog**
- organisé les cartes de contenu de manière répétable et lisible
- relu et nettoyé votre code
