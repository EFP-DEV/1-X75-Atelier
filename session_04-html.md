# Session 4 — Atelier
Durée : **4h**

Cette séance poursuit l’intégration commencée lors de la session 3.  
L’objectif est d’étendre et de consolider la structure HTML du projet, en travaillant à la fois sur les **pages publiques** et sur les **premières pages d’administration**.

Le travail reste centré sur la **structure**, la **sémantique** et la **lisibilité**.  
Le CSS n’est pas encore l’objectif de cette séance.

---

## Livrables

Vous devez rendre **deux éléments** :

1. **La poursuite ou l’amélioration des wireframes si nécessaire**
2. **Les fichiers HTML** des pages travaillées, **sans CSS**

---

## Pages à réaliser ou compléter

Vous devez poursuivre ou compléter le HTML des pages suivantes :

### Côté public
- **Home**
- **Catalog**
- **Item**

### Côté administration
- **Login**
- **Dashboard**
- **Formulaire principal de gestion**  
  (création ou édition d’un contenu)

Selon l’avancement du projet, certaines pages peuvent être encore partielles, mais leur structure principale doit exister.

---

# 1. Wireframes

Si certaines pages de la session 3 n’étaient pas suffisamment claires, vous devez corriger ou compléter les wireframes avant d’aller plus loin.

Pour les nouvelles pages d’administration, réalisez au minimum les versions suivantes :

- **Petit mobile**
- **Petite tablette**
- **Desktop**

Les wireframes doivent montrer clairement :

- la structure générale de la page
- la hiérarchie des contenus
- les zones de navigation
- les zones fonctionnelles importantes
- les formulaires et actions principales
- les différences éventuelles entre interface publique et interface d’administration

---

# 2. Intégration HTML

Pour chaque page travaillée, fournissez un fichier HTML **sans CSS**.

Le HTML doit rester :

- **sémantique**
- **accessible**
- **valide**
- conforme aux principes **WCAG**
- compréhensible **sans mise en forme CSS**

Cette séance vise à rendre le projet **plus complet** et **plus cohérent** dans son ensemble.  
On ne travaille plus uniquement le front public : on commence à structurer sérieusement l’administration.

---

## Attendus techniques

Votre code doit notamment :

- conserver une structure sémantique claire
- utiliser correctement les balises adaptées au type de contenu
- respecter une hiérarchie logique des titres
- rendre la navigation identifiable
- prévoir des formulaires compréhensibles et correctement étiquetés
- différencier clairement les zones publiques et administratives
- rester lisible, indenté et organisé
- fonctionner sans dépendre du CSS pour être compris
- préparer une base propre pour les futures étapes CSS et PHP

---

# Focus particulier par zone

## Pages publiques

Le travail doit permettre de consolider :

- la structure commune entre les pages
- les composants récurrents
- la cohérence des titres, sections et zones principales
- la lisibilité générale du contenu

Une attention particulière est attendue sur :

- l’harmonisation entre **Home**, **Catalog** et **Item**
- la stabilité des composants communs
- la qualité de la navigation

---

## Pages d’administration

Le travail doit mettre l’accent sur :

- la **page de connexion**
- le **tableau de bord**
- les **formulaires de gestion**

Les wireframes et le HTML doivent donc montrer clairement :

- une zone d’authentification identifiable
- une interface d’administration distincte du site public
- des formulaires de saisie structurés
- des actions claires : créer, modifier, enregistrer, annuler, supprimer si déjà prévu
- une navigation d’administration compréhensible

Une attention particulière est attendue sur :

- la sémantique des formulaires
- l’association correcte des `label`
- les regroupements logiques de champs
- les intitulés des boutons et actions
- la compréhension de l’interface sans CSS

---

# Critères de validation

La séance est considérée comme réussie si :

- les pages publiques continuent d’avancer proprement
- les premières pages d’administration existent
- les formulaires principaux sont structurés
- la navigation publique et la navigation admin sont identifiables
- la cohérence générale du projet progresse
- le HTML reste propre, lisible et exploitable pour la suite

---

# Récapitulatif du rendu

Vous devez fournir au minimum :

- les fichiers HTML mis à jour de **Home**, **Catalog** et **Item**
- le HTML de **Login**
- le HTML de **Dashboard**
- le HTML d’un **formulaire principal d’administration**
- les wireframes manquants ou corrigés si nécessaire

---

# Objectif de l’atelier

Cet atelier vise à :

- poursuivre l’intégration HTML du projet
- étendre le travail à l’interface d’administration
- structurer correctement les formulaires et actions de gestion
- renforcer la cohérence entre les différentes pages
- préparer une base complète avant la finalisation du HTML

---

# Checklist de fin de séance

Avant de quitter la séance, vérifiez que vous avez bien :

- poursuivi l’intégration de **Home**, **Catalog** et **Item**
- créé les premières pages HTML d’administration
- structuré une page **Login**
- structuré une page **Dashboard**
- structuré au moins un formulaire principal
- vérifié la hiérarchie des titres
- associé correctement les `label` aux champs
- rendu les actions compréhensibles
- relu et nettoyé votre code
