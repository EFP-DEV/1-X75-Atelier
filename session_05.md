# Session 5 — Atelier
Durée : **4h**

Cette séance vise à **terminer, corriger et valider** la base HTML complète du projet avant le passage au CSS.

À la fin de cette séance, le projet doit disposer d’une version statique complète et cohérente, couvrant à la fois :

- les **pages publiques**
- les **pages d’administration**
- les **formulaires principaux**
- les **éléments de navigation essentiels**

Le travail de cette séance porte sur la **finalisation du HTML**, pas sur l’habillage visuel.

---

## Livrables

Vous devez rendre :

1. **La version finalisée des fichiers HTML**
2. **Les éventuels wireframes corrigés si certaines pages ont été adaptées**
3. **Une base statique complète prête pour le CSS**

---

## Pages à finaliser

### Côté public
- **Home**
- **Catalog**
- **Item**

### Côté administration
- **Login**
- **Dashboard**
- **Formulaire principal de gestion**
- toute autre page essentielle déjà identifiée dans votre projet

Toutes les pages importantes du projet doivent maintenant exister au moins sous forme de structure HTML exploitable.

---

# 1. Finalisation des wireframes si nécessaire

Les wireframes ne sont plus le centre de la séance, mais ils peuvent être ajustés si la structure HTML réelle a évolué.

Cela concerne surtout :

- les pages ajoutées en administration
- les formulaires
- les zones de navigation
- les corrections apportées après relecture

---

# 2. Finalisation de l’intégration HTML

Pour chaque page, le HTML doit maintenant être :

- **complet**
- **propre**
- **sémantique**
- **accessible**
- **valide**
- compréhensible **sans CSS**

Le projet doit pouvoir être parcouru dans sa logique générale, même sans mise en forme graphique.

---

## Attendus techniques

Votre code doit notamment :

- utiliser une structure sémantique cohérente sur toutes les pages
- respecter une hiérarchie logique des titres
- proposer une navigation identifiable côté public et côté administration
- rendre les formulaires compréhensibles, complets et correctement étiquetés
- employer des intitulés explicites pour les actions
- rester lisible, correctement indenté et organisé
- éviter les incohérences entre pages similaires
- passer la validation HTML
- montrer une base suffisamment stable pour passer au CSS

---

# Focus particulier par zone

## Cohérence globale

Le projet doit maintenant montrer une cohérence claire entre les pages.

On attend notamment :

- une logique commune entre les pages publiques
- une logique commune entre les pages d’administration
- une distinction nette entre front et admin
- une continuité de navigation
- des composants communs identifiables

---

## Formulaires

Une attention particulière est attendue sur tous les formulaires du projet, notamment :

- formulaire de connexion
- formulaire de création ou d’édition
- filtres du catalogue
- éventuels champs de recherche ou de contact

Les formulaires doivent être :

- correctement structurés
- correctement étiquetés
- compréhensibles sans CSS
- utilisables et lisibles

---

## Validation et qualité

Cette séance doit aussi servir à corriger :

- les erreurs de structure
- les problèmes de hiérarchie des titres
- les liens mal nommés
- les boutons imprécis
- les champs sans `label`
- les incohérences de sémantique
- les oublis dans la navigation

---

# Critères de validation

La séance est considérée comme réussie si :

- toutes les pages importantes existent en HTML
- les pages publiques sont finalisées
- les pages d’administration essentielles sont présentes
- les formulaires principaux sont complets
- la navigation générale est compréhensible
- le HTML est propre, lisible et stable
- le projet est prêt à entrer en phase CSS

---

# Récapitulatif du rendu

Vous devez fournir :

- la version finalisée de tous les fichiers HTML publics
- la version finalisée des fichiers HTML d’administration
- une structure cohérente pour les formulaires principaux
- un ensemble statique complet et exploitable

À la fin de cette séance, votre projet doit disposer d’une **base HTML terminée**.

---

# Objectif de l’atelier

Cet atelier vise à :

- terminer l’intégration HTML du projet
- valider la structure de toutes les pages importantes
- corriger les incohérences de sémantique et d’accessibilité
- stabiliser les formulaires et les navigations
- préparer une transition propre vers le CSS

---

# Checklist de fin de séance

Avant de quitter la séance, vérifiez que vous avez bien :

- finalisé les pages **Home**, **Catalog** et **Item**
- finalisé les pages HTML d’administration essentielles
- vérifié tous les titres
- vérifié les liens et boutons
- vérifié les `label` de tous les formulaires
- corrigé les incohérences de structure
- validé le HTML
- relu et nettoyé l’ensemble du code
- préparé une base stable pour commencer le CSS en session 6
